Documentation

Uvis is distributed as a standalone Javascript application.

In order to use the service, you must create an HTML page and add a reference to uvis, in the page's head section:

<script src="http://uvis.dk/uvis-<version>.js"></script>

You then need to specify a minimum configuration, through a "config" object with the following fields:

  • fileProvider, a function needed by the system so that it can determinate where to find the external resources (vis and vism files) within your domain.
  • selector, a string that is the ID of the DOM element you whish the application to be injected into.
  • oData, a reference to an external OData library (not yet implemented)
      uvis.config({
        fileProvider: function (fileId) {

          // do request for filename

          return new Promise (resolve, reject) {
            resolve();
          }
        },
        selector:'',
        odata:'not yet implemented'
      });
When your configuration is ready, you can start the visEngine

uvis.run()
NOTE: If your fileProvider does not link properly to your external resources, the visEngine will fail.
